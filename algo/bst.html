Binary Search Tree (BST)

Category: Data Structures, Tree

Description: A tree data structure where each node has at most two children, with the left child containing values less than the parent and the right child containing values greater than the parent. Supports efficient search, insertion, and deletion.

📦 Business Case:

  • Sorted data storage: BSTs maintain data in sorted order allowing fast lookup, insertion, and deletion — useful in databases and file systems.
  • In-memory indexing: BSTs provide quick access to records when building indexes for search engines or in-memory caches.
  • Dynamic datasets: Suitable when data is frequently inserted or removed, such as managing user sessions or event logs.
  • Foundations for advanced trees: BSTs are the base structure for self-balancing trees (AVL, Red-Black Trees) used in high-performance systems.
🧠 How It Works?
  • Each node has up to two children, left for smaller values, right for larger.
  • Operations use binary search logic to traverse the tree efficiently.
  • Balanced BSTs maintain height to O(log n) for optimal performance.

🧮 Complexity Analysis
  • Search: O(h), where h is tree height (O(log n) average, O(n) worst)
  • Insertion: O(h)
  • Deletion: O(h)
  • Space: O(n)
