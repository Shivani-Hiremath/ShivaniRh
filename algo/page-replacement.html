Page Replacement Algorithms (LRU, LFU)

Category: Operating Systems, Caching

Description: Algorithms to decide which memory pages to evict when new pages need to be loaded.

ðŸ“¦ Business Case (Where & How to Use):

  • Operating Systems: Manage physical memory efficiently for multi-tasking systems.
  • Cloud Computing: Optimize cache management in distributed systems and CDNs.
  • Database Systems: Improve buffer cache hit rates for faster query processing.
ðŸ§  How It Works?
  • LRU (Least Recently Used): Evicts the page that has not been used for the longest time. Uses a queue or doubly linked list for O(1) updates.
  • LFU (Least Frequently Used): Evicts the page with the lowest access frequency. Uses a min-heap or frequency map for efficient tracking.
  • Both algorithms aim to minimize page faults and maximize cache hits.
ðŸ§® Complexity Analysis
  • LRU: O(1) access and update with hash map + doubly linked list
  • LFU: O(1) access and update with hash map + frequency list
  • Space: O(N), where N = cache size
