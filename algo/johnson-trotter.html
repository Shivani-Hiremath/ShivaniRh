Johnson-Trotter Algorithm

Category: Permutation Generation

Description: Generates all permutations of n elements by producing each permutation from the previous one by swapping adjacent elements.

Business Case (Amazon Ecommerce):

  • Generating all possible product bundle permutations.
  • Testing UI/UX arrangements via permutations for better customer experience.
  • Simulating recommendation scenarios with various orderings.
🧠 How It Works?
  • Assigns a direction to each element.
  • Finds the largest mobile element (one that can move in its assigned direction).
  • Swaps it with adjacent element in the direction.
  • Reverses directions of elements larger than the moved one.
  • Repeats until no mobile element exists.

🧮 Complexity Analysis
  • Time Complexity: O(n × n!) to generate all permutations
  • Space Complexity: O(n)
