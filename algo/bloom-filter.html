28. Bloom Filters

Category: Probabilistic Data Structures

Description: Space-efficient structure to test whether an element is a member of a set, allowing false positives but no false negatives.

ğŸ“¦ Business Case (Where & How to Use):

  • Cloud Storage Systems: Quickly check if data exists before fetching from disk or network (e.g., Amazon S3).
  • Database Systems: Filter out unnecessary disk reads in key-value stores.
  • Web Caches and Security: Check URL blacklists or spam filters efficiently.
ğŸ§  How It Works?
  • Initialize a bit array of size m, all set to 0.
  • Choose k independent hash functions.
  • To add an element, hash it with each function and set the corresponding bits to 1.
  • To check membership, hash the element with all k functions and check if all corresponding bits are 1.
  • If any bit is 0, the element is definitely not in the set. If all are 1, the element is probably in the set (could be a false positive).
