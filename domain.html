üìå Subset Sum

Category: Dynamic Programming

Description: Determines whether a subset of given numbers adds up to a target sum.

üì¶ Business Case:

Subset Sum using Dynamic Programming allows Amazon-like systems to efficiently make decisions like:

  • Validating gift card purchases.
  • Creating product bundles within budget.
  • Suggesting combos in recommendation systems
  • The algorithm scales efficiently across millions of products using 1D DP optimization.

üß† How It Works
  • Create a DP table: dp[i][j] where i = index, j = sum.
  • Initialize base case: dp[0][0] = true.
  • Use previous results to decide inclusion/exclusion of each item.
üìä Visualization

Given nums = {3, 34, 4, 12, 5, 2}, target sum = 9

Step-by-step DP Table Update

üë©‚Äçüíª View Code (C++) 

#include <iostream>
#include <vector>
using namespace std;

bool isSubsetSum(vector<int>& nums, int target) {
    vector<bool> dp(target + 1, false);
    dp[0] = true;
    for (int num : nums) {
        for (int j = target; j >= num; --j) {
            dp[j] = dp[j] || dp[j - num];
        }
    }
    return dp[target];
}

int main() {
    vector<int> nums = {3, 34, 4, 12, 5, 2};
    int sum = 9;
    cout << (isSubsetSum(nums, sum) ? "Found a subset" : "No subset found") << endl;
    return 0;
}
üßÆ Complexity Analysis
  • Time: O(N √ó Sum)
  • Space: O(Sum)

üîπ Edit Distance (Levenshtein Distance)

Category: Dynamic Programming

Description: Calculates the minimum number of operations (insert, delete, replace) required to convert one string to another.

üíº Business Case: Used in Amazon search engines to correct typos or suggest similar product names.

üß† How It Works
  • Create a DP table dp[i][j] where i and j represent indices in word1 and word2.
  • dp[i][j] represents the minimum operations to convert word1[0..i-1] to word2[0..j-1].
  • Base cases: converting to/from empty string.
  • Recursive case: If characters match, no operation. Else, choose min of insert, delete, replace.
üìä Visualization

Example: word1 = "kitten", word2 = "sitting"

üë©‚Äçüíª View Code (C++) 

#include <iostream>
#include <vector>
#include <string>
using namespace std;

int editDistance(string word1, string word2) {
    int m = word1.length(), n = word2.length();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1));

    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1[i - 1] == word2[j - 1])
                dp[i][j] = dp[i - 1][j - 1];
            else
                dp[i][j] = 1 + min({ dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1] });
        }
    }
    return dp[m][n];
}

int main() {
    string a = "kitten", b = "sitting";
    cout << "Edit Distance: " << editDistance(a, b) << endl;
    return 0;
}
üìä Complexity Analysis
  • Time: O(m √ó n)
  • Space: O(m √ó n)

üìå Longest Common Subsequence (LCS)

Category: Dynamic Programming

Description: Finds the length of the longest subsequence present in both strings (not necessarily contiguous).

üì¶ Business Case:

  • Used in comparing customer queries with product descriptions.
  • Helps in version control systems for diff/merge functionality.
  • Improves search relevance by matching similar patterns.
üß† How It Works
  • Use a DP table dp[i][j] where i and j index characters of the two strings.
  • If characters match, extend the LCS; otherwise take max of skipping a character from either string.
üìä Visualization

Given str1 = "AGGTAB", str2 = "GXTXAYB"

Step-by-step DP Table Update with i,j indices

üë©‚Äçüíª View Code (C++) 

#include <iostream>
#include <vector>
#include <string>
using namespace std;

int longestCommonSubsequence(string text1, string text2) {
    int n = text1.size(), m = text2.size();
    vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= m; ++j) {
            if (text1[i - 1] == text2[j - 1])
                dp[i][j] = 1 + dp[i - 1][j - 1];
            else
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
        }
    }
    return dp[n][m];
}

int main() {
    string a = "AGGTAB";
    string b = "GXTXAYB";
    cout << "Length of LCS is " << longestCommonSubsequence(a, b) << endl;
    return 0;
}
üßÆ Complexity Analysis
  • Time: O(N √ó M)
  • Space: O(N √ó M)

üéí 0/1 Knapsack Problem

Category: Dynamic Programming

Description: Given weights and profits of items and a knapsack of capacity W, find the maximum profit achievable by including or excluding items (each item can be picked at most once).

üì¶ Business Case:

  • Used in budget allocation and asset selection in finance.
  • Optimizes product placement in limited storage capacity.
  • Used in logistics for weight-restricted shipments.
üß† How It Works
  • Define dp[i][w] as the max profit for first i items and capacity w.
  • Either take the item (if it fits) or skip it ‚Äî whichever gives more profit.
üìä Visualization

Items: weights = [1, 3, 4], profits = [15, 50, 60], Capacity = 5

üë©‚Äçüíª View Code (C++) 

#include <iostream>
#include <vector>
using namespace std;

int knapsack(int W, vector<int>& weights, vector<int>& profits, int n) {
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));

    for (int i = 1; i <= n; ++i) {
        for (int w = 0; w <= W; ++w) {
            if (weights[i - 1] <= w)
                dp[i][w] = max(profits[i - 1] + dp[i - 1][w - weights[i - 1]], dp[i - 1][w]);
            else
                dp[i][w] = dp[i - 1][w];
        }
    }
    return dp[n][W];
}

int main() {
    vector<int> weights = {1, 3, 4};
    vector<int> profits = {15, 50, 60};
    int capacity = 5;
    cout << "Max profit = " << knapsack(capacity, weights, profits, weights.size()) << endl;
    return 0;
}
üßÆ Complexity Analysis
  • Time: O(N √ó W)
  • Space: O(N √ó W)

üìå Dijkstra's Algorithm

Category: Graph Algorithm (Greedy)

Description: Finds the shortest path from a source node to all other nodes in a weighted graph.

üöö Business Case Used by delivery services like Amazon or FedEx to determine the fastest delivery route.

üß† How It Works
  • Initialize distances to all vertices as infinity, except the source which is 0.
  • Use a priority queue to greedily pick the next minimum distance node.
  • Update distances of its neighbors if a shorter path is found.
  • Repeat until all nodes are visited.
